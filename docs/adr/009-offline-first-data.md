# ADR-009 資料離線優先：app 管理快取、對齊排程才更新

日期：2026-07-05｜狀態：已採納（產品擁有者要求）

## 背景

原 SW stale-while-revalidate 每次開啟都回源驗證資料。產品擁有者要求：平常
不連回伺服器；且更新判斷須對齊「每週日 18:30 UTC（台灣週一 02:30）排程部署」，
固定間隔（如 7 天）會與部署時間錯拍、常態落後。

## 決定

1. data JSON 退出 SW precache 與 runtimeCaching，改由 app 層 Cache API
   （`data-v1`）管理：首次下載入庫，之後一律讀本機。
2. 更新條件：`lastDataSync < lastScheduledUpdate()`——後者 = 最近一次
   週日 19:00 UTC（排程 18:30 + 30 分部署緩衝）。命中則背景重抓，失敗
   無感沿用舊資料；事件寫入更新紀錄（localStorage，設定頁可查）。
3. 設定頁提供「清除快取並重載」手動逃生口（清 CacheStorage + 重載）。

## 後果

- 每位使用者每週至多 1 次資料回源；離線完全可用。
- 資料時效最多落後至下次開啟 app；可接受（油價週更）。

> 修訂附註（2026-07-06）：更新條件由「本機同步時間」改為「快取油價檔的
> generatedAt」對比 lastScheduledUpdate()。原判準有洞：部署延遲時裝置會
> 同步到舊資料並蓋掉時間戳，之後整週不再重試。改判資料本身時效後，
> 拿到舊資料下次開啟仍會重試，直到新一輪上線為止（自癒）。
