# ADR-005 地圖效能：marker 視窗裁剪 + 惰性建立

日期：2026-07-04｜狀態：已採納

## 背景

Phase 1 初版把全台 1,930 站的 AdvancedMarker（每個都是 DOM 元素）一次掛上地圖，
行動裝置拖曳明顯卡頓（產品擁有者實測回報）。Google 圖磚依 ToS 不可自行快取，
效能問題須在 marker 層解決。

## 決定

1. **直營層**（626）：維持 MarkerClusterer——任何縮放下 DOM 數 = 叢集數，本來就少。
2. **加盟層**（1,313）：
   - marker **惰性建立**：只為曾進入視窗的站建 DOM（池化重用）。
   - **視窗裁剪**：僅在地圖 `idle`（拖曳結束）時增刪視窗內 marker，拖曳中零 DOM 操作。
   - 視窗內渲染上限 300（超密集市區保底）。
   - zoom < 12 整層隱藏（原規則不變）。
3. `gestureHandling: 'greedy'`＋關閉 POI 點擊，減少手勢競爭。

## 後果

- 任一時刻 DOM marker 數約 < 350（原 1,930），拖曳流暢。
- 換供應商時 culling 邏輯屬 adapter 內部實作，介面不變。
