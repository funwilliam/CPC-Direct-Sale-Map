# SPEC — 地圖搜尋覆層（search，原清單頁整併）

> 上限 120 行｜依據 PRD F5 v1.2、ADR-006｜2026-07-05 修訂

## 形態（Google Maps 慣例）

- 收合：地圖左上圓形搜尋 icon（44px FAB，白底 shadow-2）。
- 展開：頂部白色 sheet——返回鈕 + 搜尋框（自動聚焦）+ 油品篩選 chips + 結果列表
  （max-height 72%，下方地圖以 28% 黑幕弱化）。
- 點結果：關閉覆層 → 相機飛往該站（見 spec/map.md §相機移動）→ 開啟資訊卡。

## 搜尋

- fuse.js 索引欄位：`name`、`town`（1.5）、`county`、`address`；`name` 權重 2。
- threshold 0.35、ignoreLocation；輸入即搜。

## 篩選（src/lib/filter.ts 純函式）

- 油品多選 = AND（勾 98+柴油 → 兩者皆供應的站）。
- 篩選與搜尋可組合；先 fuse 搜尋再套油品條件。

## 排序與車程（ADR-006，v1.2 用量縮水）

- 搜尋中：fuse 相關性。
- 空關鍵字 + 有定位：車程排序——**開啟覆層時**才查一次 Distance Matrix，
  範圍 **10km 內最近 10 站**（v1.1 為 30km/25 站）；其餘依直線距離。
  sessionStorage 快取（~300m 格網）不重複計費。
- 無定位：直營在前，依縣市、站名。

## 顯示

- 每列：站名、直營/加盟徽章、車程「約 N 分鐘」（無車程顯示直線 km）、縣市鄉鎮、油品。
- 加盟站一眼可辨：整列淡灰底 + 紅色「加盟」徽章（PRD F5 核心要求）。
- 結果最多渲染 30 筆（覆層為快速跳轉用，非瀏覽用）。

## 驗收對照

- 搜「文山」→ 文山區各站出現；點結果 → 覆層關閉、地圖移至該站、資訊卡開啟。
- 勾 98 → 全部結果 fuels.g98 = true。
- 開啟覆層才發生 Distance Matrix 請求，且 ≤10 目的地。
